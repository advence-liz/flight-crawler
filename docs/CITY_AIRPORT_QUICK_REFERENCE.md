# 城市和机场识别速查表

## 核心概念

| 概念 | 说明 | 示例 |
|-----|------|------|
| **机场名** | 完整的机场名称 | 北京首都、上海浦东、深圳 |
| **城市名** | 机场所在的城市 | 北京、上海、深圳 |
| **后缀** | 机场名的特征部分 | 首都、浦东、宝安 |

---

## 识别流程（3 步）

```
1. 爬虫爬取航班 → origin: "北京首都", destination: "上海浦东"
                ↓
2. 提取机场名 → 机场名 = "北京首都"、"上海浦东"
                ↓
3. 提取城市名 → 城市名 = "北京"、"上海"（去掉后缀）
```

---

## 城市名提取规则

**规则**：从机场名右侧匹配已知后缀，去掉后缀即为城市名

## 数据库表结构

### Airport 表

```sql
CREATE TABLE airport (
  id INTEGER PRIMARY KEY,
  name VARCHAR(100) UNIQUE,      -- 机场名（如"北京首都"）
  city VARCHAR(50),              -- 城市名（如"北京"）
  enableCrawl BOOLEAN,           -- 是否启用爬虫
  discoveredAt TIMESTAMP,        -- 发现时间
  updatedAt TIMESTAMP            -- 更新时间
);
```

### Flight 表

```sql
CREATE TABLE flight (
  id INTEGER PRIMARY KEY,
  flightNo VARCHAR(20),
  origin VARCHAR(100),           -- 机场名（如"北京首都"）
  destination VARCHAR(100),      -- 机场名（如"上海浦东"）
  departureTime TIMESTAMP,
  arrivalTime TIMESTAMP,
  cardType VARCHAR(50),
  crawledAt TIMESTAMP
);
```

## 工作流程

### 第一次运行（发现机场）

```
1. 选择种子机场：["北京首都", "北京大兴", "上海浦东", "上海虹桥", "深圳"]
2. 从每个种子机场出发爬取航班
3. 提取 origin 和 destination
4. 发现机场：
   - 北京首都、北京大兴（北京）
   - 上海浦东、上海虹桥（上海）
   - 深圳（深圳）
   - 广州白云（广州）
   - ... 等其他目的地机场
5. 保存到 Airport 表
```

### 后续运行（刷新航班）

```
1. 从数据库查询所有启用的机场
2. 从每个机场出发爬取航班
3. 保存航班数据
4. 发现新机场（如有）
```

## 常见问题

### Q: 为什么有些机场只有一个名字？
**A**: 某些城市只有一个机场（如深圳），机场名就是城市名。后缀匹配失败时，直接返回原名。

### Q: 如何添加新的机场后缀？
**A**: 修改 `flight.service.ts` 中的 `extractCityName()` 方法，在 `airportSuffixes` 数组中添加新后缀。

### Q: 能否手动修改城市名？
**A**: 可以。在机场管理页面编辑机场信息，修改城市名。

### Q: 如何禁用某个机场？
**A**: 在机场管理页面，切换机场的启用状态开关。禁用后，爬虫不会从该机场出发。

### Q: 如何添加新的机场？
**A**: 可以手动在机场管理页面添加，或通过爬虫自动发现。
