# 爬虫日期机制说明

## 发现日期

**2026-03-17**

## 问题

为什么数据库中有多个不同日期的航班数据，但 debug 接口的 `date` 参数不生效？

## 真相

### 1. 海南航空页面的日期机制

海南航空的"随心飞"查询页面有以下特点：

- **页面默认显示"明天"的航班**
- **"明天"是相对于访问时间动态计算的**
- **没有明显的日期选择器，或日期选择器不可用**

### 2. 数据库中多日期数据的来源

数据库中的多日期航班数据是这样产生的：

| 爬取时间 | 页面默认日期 | 爬取到的航班日期 |
|---------|------------|----------------|
| 3-17 | 明天 (3-18) | 3-18 |
| 3-25 | 明天 (3-26) | 3-26 |
| 3-26 | 明天 (3-27) | 3-27 |

**结论**：这些不同日期的数据是在**不同的日期**爬取的，每次爬取都获取当时的"明天"的数据。

### 3. trigger 接口的工作方式

`trigger` 接口虽然传入了多个日期参数：

```typescript
const dates = ['2026-03-18', '2026-03-19', '2026-03-20', ...];
```

但实际上：
1. 对每个日期调用一次 `crawlFlightsByDate`
2. 每次都会重新打开页面
3. **页面始终显示"明天"的数据**
4. **传入的日期参数只用于构造返回数据的日期字段**

### 4. 日期参数的实际作用

日期参数（`date`）在代码中的作用：

```typescript
// 用于解析 API 返回的时间数据
const depTime = parseTime(
    flight.departureTime || flight.depTime || flight.takeoffTime,
    date  // ← 这里的 date 只是作为 fallback
);
```

**如果 API 返回的航班数据只包含时间（如 "08:00"），没有完整日期，则使用传入的 `date` 参数构造完整的日期时间。**

但实际上，海南航空的 API 可能返回的是完整的日期时间，所以这个 fallback 参数基本不起作用。

### 5. 为什么日期选择不生效

代码中实现了 `selectDate` 方法，但不生效的原因：

1. **页面上可能没有日期选择器**
   - 海南航空的移动端页面可能简化了UI
   - 默认只显示明天的数据

2. **日期选择器的 DOM 结构不匹配**
   - 代码中查找的选择器可能与实际页面不符
   - 导致找不到日期选择器元素

3. **代码的兜底逻辑**
   ```typescript
   if (!dateBoxClicked) {
       this.logger.warn('⚠️ 未找到日期选择器，可能页面默认日期已满足需求');
       return true; // 假设页面默认日期正确
   }
   ```
   这个兜底逻辑导致即使找不到选择器，也会继续执行，使用页面默认的"明天"。

## 影响

### 对 debug 接口的影响

- ⚠️ **`date` 参数实际上不生效**
- ⚠️ **始终爬取"明天"的数据**
- ⚠️ **无法指定特定日期进行调试**

### 对 trigger 接口的影响

- ⚠️ **传入的日期参数实际上不生效**
- ⚠️ **每次爬取都获取"明天"的数据**
- ✅ **但由于是定时任务，每天执行，所以能够累积不同日期的数据**

### 对数据完整性的影响

- ⚠️ **只能获取"明天"的数据**
- ⚠️ **无法一次性爬取未来 7 天的数据**
- ⚠️ **需要依赖定时任务每天执行，逐步累积数据**

## 解决方案

### 短期方案

1. **接受现状**
   - 依赖定时任务每天执行
   - 逐步累积未来数据

2. **调整预期**
   - debug 接口只能调试"明天"的数据
   - trigger 接口虽然传入多个日期，但实际只爬取"明天"

### 长期方案

1. **分析页面实际结构**
   - 使用浏览器开发者工具分析页面
   - 确认是否真的有日期选择器
   - 如果有，找到正确的选择器

2. **完善 `selectDate` 方法**
   - 更新选择器匹配逻辑
   - 实现真正的日期选择功能

3. **考虑使用 URL 参数**
   - 检查海南航空 API 是否支持日期参数
   - 如果支持，直接在 URL 中传入日期

## 建议

### 对于开发者

- 不要依赖 `date` 参数来指定爬取日期
- 理解爬虫实际上只能爬取"明天"的数据
- 依赖定时任务来累积不同日期的数据

### 对于用户

- 使用 debug 接口时，不要指定 `date` 参数
- 理解返回的数据是"明天"的数据
- 如需完整的未来数据，等待定时任务逐日累积
