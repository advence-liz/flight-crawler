# 数据管理功能说明

## 🎯 功能概述

新增独立的"数据管理"页面，将数据爬取功能从目的地查询页面分离出来，提供更专业和友好的数据管理界面。

### 核心改进

1. ✅ **独立管理界面**：专门的数据管理页面，清晰的功能分区
2. ✅ **自定义时间范围**：支持用户自定义爬取天数
3. ✅ **可视化流程**：使用 Steps 组件展示执行流程
4. ✅ **实时反馈**：显示执行结果和统计数据
5. ✅ **使用指导**：详细的使用说明和注意事项

---

## 📊 功能模块

### 1. 一键初始化（推荐）

**功能描述**：
- 自动执行"发现机场"和"发现航班"两个步骤
- 使用默认配置（发现机场 1 天，发现航班 7 天）
- 适合首次使用和快速初始化

**使用场景**：
- 首次部署系统，需要快速获取数据
- 定期全量更新数据

### 2. 发现机场（步骤 1）

**功能描述**：
- 从种子机场爬取航班数据
- 自动发现所有可用的出发地和目的地机场
- 支持自定义爬取天数（1-7 天）

**参数配置**：
- **爬取天数**：1-7 天（推荐 1 天）
- **默认值**：1 天

**输出结果**：
- 发现的机场数量
- 临时航班数量

### 3. 发现航班（步骤 2）

**功能描述**：
- 基于已发现的机场，全量爬取航班数据
- 支持自定义爬取天数（1-30 天）
- 自动清理旧数据，确保数据最新

**参数配置**：
- **爬取天数**：1-30 天（推荐 7-30 天）
- **默认值**：7 天

**输出结果**：
- 爬取的航班总数

## 🔧 技术实现

### 后端改动

#### 1. 新增 DTO

**文件**: `backend/src/modules/crawler/dto/initialize.dto.ts`

```typescript
export class InitializeDiscoverDto {
  @IsOptional()
  @Type(() => Number)
  @IsInt()
  @Min(1)
  @Max(7)
  days?: number = 1; // 默认 1 天
}

export class InitializeRefreshDto {
  @IsOptional()
  @Type(() => Number)
  @IsInt()
  @Min(1)
  @Max(30)
  days?: number = 7; // 默认 7 天
}
```

## 📋 使用指南

### 首次使用

1. 访问"数据管理"页面
2. 点击"开始一键初始化"按钮
3. 等待执行完成（约 5-10 分钟）
4. 查看统计结果
5. 前往"目的地查询"页面开始使用

### 定期更新

**推荐频率**：每天执行一次

**方式一：一键初始化**
- 适合全量更新数据
- 使用默认配置即可

**方式二：仅更新航班**
- 进入"步骤 2：发现航班"
- 设置爬取天数（如 30 天）
- 点击"开始发现航班"

### 自定义时间范围

**场景 1：快速测试**
- 发现机场：1 天
- 发现航班：1 天
- 执行时间：约 2-3 分钟

**场景 2：标准配置（推荐）**
- 发现机场：1 天
- 发现航班：7 天
- 执行时间：约 5-10 分钟

**场景 3：完整数据**
- 发现机场：1 天
- 发现航班：30 天
- 执行时间：约 15-30 分钟

## 🔍 API 示例

### 1. 发现机场（自定义天数）

**请求**:
```bash
POST /api/crawler/initialize/discover
Content-Type: application/json

{
  "days": 3
}
```

**响应**:
```json
{
  "success": true,
  "airportCount": 15,
  "flightCount": 120,
  "message": "机场发现完成！发现 15 个机场，120 条航班"
}
```

### 2. 发现航班（自定义天数）

**请求**:
```bash
POST /api/crawler/initialize/refresh
Content-Type: application/json

{
  "days": 30
}
```

**响应**:
```json
{
  "success": true,
  "count": 3500,
  "message": "航班发现完成！共爬取 3500 条航班"
}
```

### 3. 使用默认值

**请求**（不传参数）:
```bash
POST /api/crawler/initialize/discover
Content-Type: application/json

{}
```

**响应**（使用默认值 1 天）:
```json
{
  "success": true,
  "airportCount": 15,
  "flightCount": 40,
  "message": "机场发现完成！发现 15 个机场，40 条航班"
```

## ⚠️ 注意事项

### 执行顺序

1. **首次使用**：必须先执行"发现机场"，再执行"发现航班"
2. **后续更新**：如果机场列表已存在，可以直接执行"发现航班"

### 时间范围建议

| 功能 | 推荐天数 | 最小值 | 最大值 | 说明 |
|------|---------|--------|--------|------|
| 发现机场 | 1 天 | 1 天 | 7 天 | 1 天足够发现所有机场 |
| 发现航班 | 7-30 天 | 1 天 | 30 天 | 天数越多，数据越完整 |

### 执行时间估算

- **发现机场（1 天）**：约 1-2 分钟
- **发现航班（7 天）**：约 5-10 分钟
- **发现航班（30 天）**：约 15-30 分钟

实际时间取决于：
- 网络速度
- 服务器响应速度
- 机场数量
- 航班数量

### 数据更新策略

- **发现机场**：只清理临时数据，不影响已有航班数据
- **发现航班**：清理所有未来的航班数据，然后重新爬取

### 频率控制

- **建议频率**：每天执行一次
- **避免频繁执行**：过于频繁的爬取可能触发反爬机制
- **错峰执行**：建议在凌晨或访问低峰期执行
