## 技术实现方案

### 方案 1: Puppeteer 动态爬虫（推荐）
**优点**:
- 完整模拟浏览器行为
- 可处理 JavaScript 渲染
- 支持交互式操作（点击、输入）
- 可截图调试

**缺点**:
- 资源消耗较大
- 速度相对较慢

**实现步骤**:
```javascript
1. 启动 Puppeteer 浏览器实例
2. 访问目标 URL
3. 等待页面加载完成
4. 选择出发地（触发下拉菜单）
5. 选择目的地
6. 选择日期范围
7. 点击查询按钮
8. 等待结果加载
9. 提取航班数据（价格、航班号、时间等）
10. 保存到数据库
```

### 方案 2: API 逆向（高效但需要分析）
**流程**:
1. 使用浏览器开发者工具（Network 面板）
2. 监控查询时发送的 HTTP 请求
3. 分析请求参数和响应格式
4. 直接调用 API 获取数据

**优点**:
- 速度快
- 资源消耗小
- 易于批量处理

**缺点**:
- 需要逆向分析
- API 可能有反爬机制（签名、加密）
- API 结构变化需要更新代码

### 方案 3: 混合方案
- 初期使用 Puppeteer 进行原型开发和调试
- 分析出 API 后切换到直接调用
- 保留 Puppeteer 作为备用方案

## 反爬虫应对策略

### 1. 请求头伪装
```javascript
headers: {
  'User-Agent': 'Mozilla/5.0 (iPhone; CPU iPhone OS 14_0 like Mac OS X)...',
  'Accept-Language': 'zh-CN,zh;q=0.9',
  'Referer': 'https://m.hnair.com/'
}
```

### 2. 请求频率控制
- 随机延迟: 500ms - 2000ms
- 每日请求限制: < 1000 次
- 分时段爬取（避开高峰期）

### 3. IP 代理池（可选）
- 使用代理服务
- 轮换 IP 地址
- 避免单一 IP 被封禁

### 4. Cookie 和 Session 管理
- 保持会话一致性
- 定期更新 Cookie
- 模拟真实用户行为

interface Flight {

## 错误处理

### 常见错误
1. **网络超时**: 重试机制（最多 3 次）
2. **页面结构变化**: 发送告警通知
3. **验证码**: 人工介入或使用打码服务
4. **IP 被封**: 切换代理或暂停爬取
