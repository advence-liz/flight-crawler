### 步骤 2：清理卡住的任务 ✅

#### 问题发现
发现任务 ID 11 从昨天 17:35 开始运行，已运行约 8 小时，但 flightCount 仍为 0。

**原因分析**：
- 任务是在后端重启前启动的
- 后端重启后，任务状态不一致
- 内存中的并发锁状态已清空，但数据库中的任务状态仍为 "running"

#### 清理操作
```sql
UPDATE crawler_logs
SET status = 'failed',
    errorMessage = '任务超时，已手动清理',
    endTime = datetime('now'),
    duration = 28800
WHERE id = 11 AND status = 'running';
```

**结果**：
- ✅ 任务状态已更新为 "failed"
- ✅ 添加了错误信息说明
- ✅ 记录了执行时长（8 小时）
