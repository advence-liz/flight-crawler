# 城市和机场识别流程总结

## 📌 一句话总结

爬虫从航班数据中自动提取**机场名**，再通过**后缀匹配**提取**城市名**。

---

## 🔄 完整流程

### 1️⃣ 爬虫爬取阶段

**输入**：用户指定的出发机场（种子机场）
```
种子机场: 北京首都、上海浦东、深圳
```

**过程**：
- 访问海南航空随心飞页面
- 选择出发机场和日期
- 爬取所有可达目的地的航班

**输出**：航班数据
```
航班1: origin="北京首都", destination="上海浦东"
航班2: origin="北京首都", destination="深圳"
航班3: origin="北京首都", destination="广州白云"
...
```

### 2️⃣ 机场发现阶段

**输入**：爬虫爬取的航班数据

**过程**：
```typescript
// 遍历所有航班
for each flight in flights {
  // 从 origin 提取机场
  airport1 = flight.origin                    // "北京首都"
  city1 = extractCityName(airport1)           // "北京"
  save(airport1, city1)

  // 从 destination 提取机场
  airport2 = flight.destination              // "上海浦东"
  city2 = extractCityName(airport2)           // "上海"
  save(airport2, city2)
}
```

**输出**：机场列表
```
机场1: name="北京首都", city="北京"
机场2: name="上海浦东", city="上海"
机场3: name="深圳", city="深圳"
机场4: name="广州白云", city="广州"
...
```

### 3️⃣ 数据保存阶段

**输入**：发现的机场列表

**过程**：保存到 Airport 表

**输出**：数据库中的机场数据

## 🎯 城市名提取规则

### 核心规则

```
城市名 = 机场名 - 后缀
```

### 匹配过程

```
机场名: "北京首都"
       ↓
检查是否以"首都"结尾？ → YES
       ↓
城市名 = "北京首都" - "首都" = "北京"
```

### 后缀列表（19 个）

```
首都、大兴、浦东、虹桥、白云、宝安、天河、萧山、流亭、周水子、
江北、双流、天府、咸阳、长水、黄花、凤凰、新、机场
```

### 特殊情况

**无后缀的机场**：返回原名
```
机场名: "深圳"
检查所有后缀都不匹配
→ 城市名 = "深圳"
```

## 🔑 关键特性

### ✅ 自动发现
- 无需手动配置所有机场
- 从爬虫数据中自动提取
- 支持动态扩展

### ✅ 智能提取
- 基于已知后缀列表
- 支持同城多机场（北京首都、北京大兴）
- 无后缀的机场也能识别（深圳）

### ✅ 去重机制
- 使用 Set 数据结构
- 同一机场多次发现只保存一次

### ✅ 启用/禁用控制
- 每个机场都有 enableCrawl 标志
- 爬虫只从启用的机场出发
- 可通过 UI 快速切换

## 🚀 工作流程

### 发现机场工作流

```
开始
  ↓
选择种子机场（5个）
  ↓
并行爬取所有种子机场
  ↓
保存航班数据到 Flight 表
  ↓
遍历所有航班，发现机场
  ↓
提取城市名（后缀匹配）
  ↓
保存机场到 Airport 表
  ↓
完成
```

### 刷新航班工作流

```
开始
  ↓
查询所有启用的机场
  ↓
从每个机场出发爬取航班
  ↓
保存航班数据
  ↓
发现新机场（如有）
  ↓
完成
```

## ❓ 常见问题

### Q1: 为什么某些机场没有后缀？
**A**: 某些城市只有一个机场（如深圳、重庆），机场名就是城市名。

### Q2: 如何添加新的机场后缀？
**A**: 修改 `flight.service.ts` 的 `extractCityName()` 方法，在 `airportSuffixes` 数组中添加。

### Q3: 能否手动修改城市名？
**A**: 可以。在机场管理页面编辑机场信息。

### Q4: 如何禁用某个机场？
**A**: 在机场管理页面切换启用状态开关。

### Q5: 如何手动添加机场？
**A**: 在机场管理页面点击添加按钮，或通过爬虫自动发现。
