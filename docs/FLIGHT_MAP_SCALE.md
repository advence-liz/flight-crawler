# 航线地图元素缩放优化说明

## 问题描述

航线地图中的节点、连线、文字等元素体积较大，在某些场景下（特别是目的地较多时）可能导致：
1. **节点过大**：出发地、目的地节点占用过多空间
2. **连线过粗**：航线连线显得拥挤
3. **文字过大**：节点标签和 Tooltip 文字占用空间
4. **整体拥挤**：多个目的地时图表显得拥挤，难以查看

**优化需求**：将航线图中所有元素体积整体缩小 30%

---

## 解决方案

对航线图中的所有视觉元素进行**统一缩放**，将尺寸缩小到原来的 70%（即缩小 30%）：
- **节点大小**：symbolSize 缩小 30%
- **连线宽度**：lineStyle.width 缩小 30%
- **箭头大小**：symbolSize 缩小 30%
- **文字大小**：fontSize 缩小 30%
- **边框宽度**：borderWidth 缩小 30%
- **间距距离**：distance 缩小 30%

## 实现细节

### 1. 出发地节点缩放

**文件路径**：`frontend/src/pages/FlightMap.tsx`（第 136-147 行）

**修改前**：
```typescript
{
  id: origin,
  name: origin,
  symbolSize: 48,  // 节点大小
  itemStyle: { color: '#facc15' },
  label: {
    show: true,
    color: '#1a1a1a',
    fontWeight: 700,
    fontSize: 13  // 文字大小
  },
  // ...
}
```

**修改后**：
```typescript
{
  id: origin,
  name: origin,
  symbolSize: 34,  // 48 * 0.7 ≈ 34 (缩小 30%)
  itemStyle: { color: '#facc15' },
  label: {
    show: true,
    color: '#1a1a1a',
    fontWeight: 700,
    fontSize: 9  // 13 * 0.7 ≈ 9 (缩小 30%)
  },
  // ...
}
```

**缩放比例**：
- 节点大小：48 → 34（缩小 29%）
- 文字大小：13 → 9（缩小 31%）

### 2. 直飞目的地节点缩放

**文件路径**：`frontend/src/pages/FlightMap.tsx`（第 154-205 行）

**修改前**：
```typescript
{
  symbolSize: Math.max(20, Math.min(38, dest.flightCount * 1.2 + 14)),
  itemStyle: {
    borderWidth: 1.5,
  },
  label: { fontSize: 11 },
}

// 去程连线
{
  lineStyle: {
    width: isReturn ? 2 : 1,
  },
  symbolSize: [0, 7],  // 箭头大小
}

// 返程连线
{
  lineStyle: { width: 1.5 },
  symbolSize: [0, 6],
}
```

**修改后**：
```typescript
{
  symbolSize: Math.max(14, Math.min(27, dest.flightCount * 0.84 + 10)),
  // 计算说明：
  // - 最小值：20 * 0.7 = 14
  // - 最大值：38 * 0.7 ≈ 27
  // - 系数：1.2 * 0.7 = 0.84
  // - 基数：14 * 0.7 ≈ 10
  itemStyle: {
    borderWidth: 1,  // 1.5 * 0.7 ≈ 1
  },
  label: { fontSize: 8 },  // 11 * 0.7 ≈ 8
}

// 去程连线
{
  lineStyle: {
    width: isReturn ? 1.4 : 0.7,  // 2→1.4, 1→0.7
  },
  symbolSize: [0, 5],  // 7 * 0.7 ≈ 5
}

// 返程连线
{
  lineStyle: { width: 1 },  // 1.5 * 0.7 ≈ 1
  symbolSize: [0, 4],  // 6 * 0.7 ≈ 4
}
```

**缩放比例**：
- 节点大小：20-38 → 14-27（缩小 30%）
- 边框宽度：1.5 → 1（缩小 33%）
- 文字大小：11 → 8（缩小 27%）
- 去程连线：2/1 → 1.4/0.7（缩小 30%）
- 返程连线：1.5 → 1（缩小 33%）
- 箭头大小：7/6 → 5/4（缩小 29-33%）

## 改进效果

### 视觉效果提升

#### 修改前 ❌
- 节点较大，多个目的地时显得拥挤
- 连线较粗，交叉时难以区分
- 文字较大，标签容易重叠
- 整体显得"沉重"，视觉压迫感强

#### 修改后 ✅
- 节点适中，多个目的地时布局更清晰
- 连线精细，交叉时容易区分
- 文字适中，标签重叠减少
- 整体显得"轻盈"，视觉舒适度提升

### 实际应用场景

**少量目的地（< 10 个）**：
- 修改前：节点较大，但布局清晰
- 修改后：节点适中，布局更优雅

**中等目的地（10-20 个）**：
- 修改前：节点开始拥挤，标签可能重叠
- 修改后：节点分布合理，标签重叠减少

**大量目的地（> 20 个）**：
- 修改前：节点严重拥挤，连线交叉混乱
- 修改后：节点布局清晰，连线区分明显

## 技术实现细节

### 1. 动态节点大小计算

**直飞目的地节点**：
```typescript
symbolSize: Math.max(14, Math.min(27, dest.flightCount * 0.84 + 10))
```

**计算逻辑**：
- 基础大小：10（原 14，缩小 30%）
- 航班系数：0.84（原 1.2，缩小 30%）
- 最小尺寸：14（原 20，缩小 30%）
- 最大尺寸：27（原 38，缩小 30%）

**示例**：
- 5 班航班：10 + 5 * 0.84 = 14.2 → 14（最小值限制）
- 10 班航班：10 + 10 * 0.84 = 18.4
- 15 班航班：10 + 15 * 0.84 = 22.6
- 20 班航班：10 + 20 * 0.84 = 26.8 → 27（最大值限制）

### 2. 连线宽度分级

| 连线类型 | 原宽度 | 新宽度 | 说明 |
|---------|-------|-------|------|
| 往返去程 | 2 | 1.4 | 最粗，强调往返 |
| 单程去程 | 1 | 0.7 | 较细，普通航线 |
| 返程 | 1.5 | 1 | 中等，虚线 |
| 中转 | 1 | 0.7 | 较细，虚线 |

**视觉层级**：往返 > 返程 > 单程 = 中转

### 3. 文字大小分级

| 文字类型 | 原大小 | 新大小 | 说明 |
|---------|-------|-------|------|
| 出发地标签 | 13 | 9 | 最大，强调出发地 |

| 中转标签 | 10 | 7 | 较小，次要信息 |

## 未来优化建议

如果需要进一步优化航线图，可以考虑：

1. **响应式缩放**：根据屏幕尺寸动态调整元素大小
   ```typescript
   const scale = isMobile ? 0.6 : 1;
   symbolSize: 48 * scale
   ```

2. **密度自适应**：根据目的地数量动态调整元素大小
   ```typescript
   const scale = Math.max(0.5, 1 - destinations.length / 100);
   ```

3. **分层渲染**：目的地过多时只渲染重要节点，其他节点缩小或隐藏

4. **聚合节点**：相近的目的地聚合为一个节点，点击展开

5. **自定义缩放级别**：提供用户可调节的缩放比例滑块
