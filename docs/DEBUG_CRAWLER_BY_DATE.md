# 按日期调试爬虫功能

## 功能概述

新增的 `debugFlightByDate` 功能用于诊断特定出发地和日期的航班缺失问题。该功能会爬取指定条件的航班数据，但**不保存到数据库**，仅用于调试和诊断。

## API 端点

```
GET /api/crawler/debug?origin=北京首都&date=2026-03-19
```

## 参数说明

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| origin | string | 否 | 北京 | 出发地（如：北京首都、上海浦东） |
| date | string | 否 | 明天 | 日期（格式：YYYY-MM-DD）⚠️ **当前版本限制：由于海南航空页面的日期选择机制，目前只能爬取明天的数据** |

## 返回结果

```typescript
{
  success: boolean;        // 是否成功
  count: number;           // 找到的航班数量
  flights: Flight[];       // 航班详细数据
  screenshots: string[];   // 本次调试生成的截图路径
  logs: string[];          // 相关日志（最多 50 条）
}
```

## 使用场景

### 1. 诊断特定航线缺失问题

**问题**：用户反馈"北京首都→乌鲁木齐"航线在 2026-03-19 缺失

**解决步骤**：

1. 调用调试接口
```bash
curl "http://localhost:3000/api/crawler/debug?origin=北京首都&date=2026-03-19"
```

2. 查看返回结果
   - `count`: 该日期爬取到的航班数量
   - `flights`: 查看是否包含乌鲁木齐航班
   - `screenshots`: 查看截图确认页面数据
   - `logs`: 查看日志确认过滤原因

3. 分析原因
   - 如果 `count = 0`：页面可能没有数据或爬虫失败
   - 如果 `flights` 中没有乌鲁木齐：检查 `logs` 查看过滤原因
   - 查看 `screenshots` 确认页面实际显示的数据

### 2. 验证爬虫逻辑

**场景**：修改了爬虫逻辑后，需要验证是否正常工作

**步骤**：

1. 不提供日期参数（默认爬取明天）
```bash
curl "http://localhost:3000/api/crawler/debug?origin=上海浦东"
```

2. 查看返回的 `flights` 数据是否符合预期

3. 检查 `logs` 确认过滤逻辑是否正确

### 3. 对比不同出发地

**场景**：对比北京首都和北京大兴的航班数据

**步骤**：

1. 爬取北京首都
```bash
curl "http://localhost:3000/api/crawler/debug?origin=北京首都&date=2026-03-19"
```

2. 爬取北京大兴
```bash
curl "http://localhost:3000/api/crawler/debug?origin=北京大兴&date=2026-03-19"
```

3. 对比两个结果的 `count` 和 `flights`

## 功能特点

### 1. 不影响数据库
- ✅ 调试数据**不会保存到数据库**
- ✅ 不会影响现有数据
- ✅ 可以重复调试同一日期

### 2. 自动收集诊断信息
- ✅ 自动收集本次调试的截图
- ✅ 自动提取相关日志（最后 50 条）
- ✅ 返回完整的航班数据

### 3. 灵活的参数
- ✅ 不提供日期则默认爬取明天
- ✅ 不提供出发地则默认北京
- ✅ 支持任意日期和出发地组合

## 注意事项

1. **⚠️ 日期选择限制（重要）**
   - **当前版本只能爬取明天的数据**
   - 海南航空页面默认显示明天的航班
   - 自定义日期选择功能尚未完全实现
   - 即使传入 `date` 参数，返回的仍是明天的数据
   - 如需爬取其他日期，建议直接修改系统时间或等待功能完善

2. **不保存到数据库**
   - 调试数据仅返回，不会保存
   - 需要正式爬取请使用 `/api/crawler/initialize/refresh`

3. **截图和日志**
   - 截图保存在 `backend/debug-screenshots/` 目录
   - 日志保存在 `backend/logs/` 目录
   - 只返回本次调试相关的截图和日志

4. **性能考虑**
   - 调试会启动浏览器，耗时较长（30-60 秒）
   - 建议在开发环境使用
   - 避免频繁调用

5. **日期格式**
   - 必须使用 ISO 格式：`YYYY-MM-DD`
   - 例如：`2026-03-19`
   - ⚠️ 但目前只返回明天的数据
