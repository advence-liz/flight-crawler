## 1. 项目概述

### 1.1 项目背景
随心飞产品提供特价机票，但用户难以快速了解所有可达目的地和最优中转方案。本工具通过爬虫技术自动获取航班数据，并提供智能分析和行程规划功能。

### 1.2 项目目标
- 自动化获取随心飞特价航班数据
- 提供直观的 Web 界面进行行程查询和规划
- 支持多城市中转的智能路径规划

---

## 2. 功能需求

### 2.1 核心功能

#### 功能 1: 目的地查询
**功能描述**: 根据起点和时间范围，查询所有可到达的目的地

**输入参数**:
- 起点城市 (必填): 如 "北京"、"上海"
- 开始日期 (必填): YYYY-MM-DD 格式
- 结束日期 (必填): YYYY-MM-DD 格式
- 航班类型 (可选): 直飞/含中转

**输出结果**:
- 目的地城市列表
- 每个目的地的最低价格
- 可用日期列表
- 航班数量统计

**界面要求**:
- 表格形式展示结果
- 支持按价格、日期排序
- 支持导出为 Excel/CSV
- 展示价格趋势图表

**示例**:
```
输入: 起点=北京, 日期范围=2026-04-01 至 2026-04-30
输出:
  - 上海: ¥299 (15个航班)
  - 成都: ¥399 (8个航班)
  - 广州: ¥499 (12个航班)
  ...
```

---

#### 功能 2: 智能行程规划
**功能描述**: 根据起点、终点和时间，规划最优行程（支持中转）

**输入参数**:
- 起点城市 (必填)
- 终点城市 (必填)
- 出发日期 (必填)
- 返程日期 (可选)
- 最大中转次数 (默认: 2次)
- 中转最短停留时间 (默认: 2小时)
- 中转最长停留时间 (默认: 24小时)
- 价格排序优先级 (可选): 最低价/最少中转/最短时间

**输出结果**:
- 推荐行程方案列表 (Top 10)
- 每个方案包含:
  - 总价格
  - 总耗时
  - 中转城市和停留时间
  - 各段航班详情 (航班号、起降时间、价格)
  - 可行性评分

**高级功能**:
- 支持指定中转城市 (如: "想在成都停留游玩")
- 支持多目的地串联 (如: 北京 → 成都 → 重庆 → 北京)
- 价格预警: 当价格低于设定阈值时推送通知

**界面要求**:
- 时间轴可视化展示行程
- 地图展示飞行路线
- 支持方案对比功能
- 一键保存行程到收藏

**示例**:
```
输入: 北京 → 三亚, 2026-05-01出发, 最多1次中转
输出方案1:
  北京 → 广州 (CA1234, 08:00-11:30, ¥299)
  [停留 3小时]
  广州 → 三亚 (CZ5678, 14:30-16:00, ¥199)
  总价: ¥498, 总耗时: 8小时
```

## 3. 非功能需求

### 3.1 性能要求
- 查询响应时间 < 3秒
- 支持并发 50+ 用户
- 爬虫单次任务完成时间 < 10分钟

### 3.2 数据要求
- 数据更新频率: 每日至少1次
- 数据保留周期: 6个月
- 支持历史价格对比

### 3.3 兼容性
- 浏览器: Chrome/Edge/Safari 最新版本
- 响应式设计: 支持移动端访问

### 3.4 安全性
- 爬虫请求频率控制 (防止被封禁)
- 用户数据加密存储
- API 接口鉴权

## 4. 技术实现方案

### 4.1 爬虫策略
- 使用 Puppeteer 模拟浏览器访问
- 请求头伪装 + 随机延迟
- IP 代理池 (可选)
- 增量更新策略

### 4.2 路径规划算法
- 基于图论的最短路径算法 (Dijkstra/A*)
- 考虑时间窗口约束
- 多目标优化 (价格/时间/中转次数)

### 4.3 数据存储
```sql
-- 航班表
flights (
  id, flight_no, origin, destination,
  departure_time, arrival_time, price,
  available_seats, crawled_at
)

-- 用户查询历史
search_history (
  id, user_id, origin, destination,
  date_range, created_at
)

-- 收藏行程
saved_routes (
  id, user_id, route_data, created_at
)
```

## 6. 风险与挑战

### 6.1 技术风险
- **反爬虫机制**: 目标网站可能有验证码、IP 限制
  - 应对: 使用代理池、请求频率控制、验证码识别

- **数据准确性**: 价格和余票实时变化
  - 应对: 标注数据更新时间、提供"实时验证"功能

### 6.2 合规风险
- **数据使用合规**: 确保爬虫行为符合目标网站 robots.txt
- **用户隐私**: 遵守数据保护法规
