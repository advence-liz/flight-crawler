# 日志表创建问题修复

## 🐛 问题描述

执行"发现机场"或"发现航班"后，日志没有保存到数据库中。

## 🔍 问题原因

虽然代码实现正确，但数据库中缺少 `crawler_logs` 表。

**原因分析**：
1. TypeORM 的 `synchronize: true` 配置在**服务启动时**才会扫描实体并创建表
2. 新增 `CrawlerLog` 实体后，如果后端服务没有重启，表不会被自动创建
3. 即使编译成功（`npm run build`），也需要重启服务才能生效

## ✅ 解决方案

### 方案 1：手动创建表（已执行）✅

使用提供的 SQL 脚本手动创建表：

```bash
sqlite3 backend/data/flight-crawler.db < backend/create-crawler-logs-table.sql
```

**优点**：
- 立即生效，无需重启服务
- 适合开发环境快速修复

**已创建的表**：
```sql
CREATE TABLE crawler_logs (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  taskType VARCHAR(50) NOT NULL,
  status VARCHAR(20) NOT NULL,
  days INTEGER,
  airportCount INTEGER DEFAULT 0,
  flightCount INTEGER DEFAULT 0,
  details TEXT,
  errorMessage TEXT,
  startTime DATETIME,
  endTime DATETIME,
  duration INTEGER,
  createdAt DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_taskType_createdAt ON crawler_logs(taskType, createdAt);
CREATE INDEX idx_status_createdAt ON crawler_logs(status, createdAt);
```

### 方案 2：重启后端服务

重启后端服务，让 TypeORM 自动创建表：

```bash
# 停止当前服务
Ctrl+C

# 重新启动
cd backend
npm run dev
```

**优点**：
- 符合 TypeORM 的正常工作流程
- 生产环境推荐方式

## ⚠️ 注意事项

### 1. 开发环境

- 添加新实体后，需要重启后端服务
- 或者手动创建表（使用 SQL 脚本）

### 2. 生产环境

- **强烈建议关闭 `synchronize: true`**
- 使用数据库迁移脚本管理表结构
- 避免自动同步导致数据丢失

### 3. TypeORM 自动同步的工作原理

```typescript
// app.module.ts
TypeOrmModule.forRoot({
  type: 'sqlite',
  database: './data/flight-crawler.db',
  entities: [__dirname + '/**/*.entity{.ts,.js}'], // 扫描所有实体
  synchronize: true, // 启动时自动同步表结构
  logging: process.env.NODE_ENV === 'development',
})
```

**执行时机**：
- 仅在**应用启动时**执行一次
- 扫描 `entities` 路径下的所有实体文件
- 对比数据库表结构，自动创建/更新表

**限制**：
- 不会在运行时监听实体文件变化
- 添加新实体后必须重启应用

## 🚀 后续建议

### 1. 使用数据库迁移

生产环境应使用 TypeORM 迁移功能：

```bash
# 生成迁移文件
npm run typeorm migration:generate -- -n CreateCrawlerLogsTable

# 执行迁移
npm run typeorm migration:run
```

### 2. 关闭自动同步

生产环境配置：

```typescript
TypeOrmModule.forRoot({
  type: 'postgres', // 生产环境使用 PostgreSQL
  synchronize: false, // 关闭自动同步
  migrations: ['dist/migrations/*{.ts,.js}'],
  migrationsRun: true, // 启动时自动运行迁移
})
```

### 3. 添加健康检查

检测表是否存在：

```typescript
async onModuleInit() {
  const tableExists = await this.dataSource.query(
    "SELECT name FROM sqlite_master WHERE type='table' AND name='crawler_logs'"
  );

  if (!tableExists.length) {
    this.logger.error('❌ crawler_logs 表不存在，请重启服务或手动创建表');
  }
}
```
