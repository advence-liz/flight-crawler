## 🎉 新增功能

### 往返航班查询与展示

在目的地查询页面中，新增了完整的往返航班信息展示功能。

#### 主要特性

1. **目的地列表增强**
   - ✅ 显示"可往返"标签，一眼识别支持往返的目的地
   - ✅ 展示去程航班数量和可用日期天数
   - ✅ 展示返程航班数量和可用日期天数
   - ✅ 无返程时显示"无返程"提示

2. **往返航班详情**
   - ✅ 分段展示去程和返程航班
   - ✅ 清晰的标题：`起点 ⇄ 目的地 - 往返航班详情`
   - ✅ 每个航班显示航班号、起降时间、权益卡类型
   - ✅ 无返程时友好提示

3. **智能查询**
   - ✅ 自动查询返程航班信息
   - ✅ 应用相同的日期范围和权益卡类型过滤
   - ✅ 一次请求获取完整往返信息

## 🔧 技术实现

### 后端改动

#### 1. 数据模型扩展

**文件**: `backend/src/modules/flight/dto/destination-result.dto.ts`

```typescript
export class DestinationResultDto {
  // 原有字段
  destination: string;
  flightCount: number;
  availableDates: string[];
  cardTypes: string[];

  // 新增字段
  hasReturn: boolean;              // 是否有返程航班
  returnFlightCount?: number;      // 返程航班数量
  returnAvailableDates?: string[]; // 返程可用日期
}

export class RoundTripFlightsDto {
  outboundFlights: Flight[];  // 去程航班列表
  returnFlights: Flight[];    // 返程航班列表
}
```

#### 2. Service 方法增强

**文件**: `backend/src/modules/flight/flight.service.ts`

- **`queryDestinations()`**: 增强逻辑，自动查询返程航班信息
- **`queryRoundTripFlights()`**: 新增方法，同时查询去程和返程航班详情

#### 3. 新增 API 端点

**文件**: `backend/src/modules/flight/flight.controller.ts`

```
GET /api/flights/round-trip
参数：origin, destination, startDate, endDate, flightType
响应：{ outboundFlights: [...], returnFlights: [...] }
```

### 前端改动

#### 1. API 层扩展

**文件**: `frontend/src/api/flight.ts`

- 更新 `DestinationResult` 接口，添加往返字段
- 新增 `RoundTripFlights` 接口
- 新增 `queryRoundTripFlights()` 方法

#### 2. UI 组件改进

**文件**: `frontend/src/pages/DestinationQuery.tsx`

- 表格新增"返程航班"列
- 目的地列显示"可往返"标签
- Modal 分段展示去程和返程航班
- 优化视觉呈现和用户体验

## 📝 API 使用示例

### 1. 查询目的地（包含往返信息）

**请求**:
```bash
GET /api/flights/destinations?origin=北京首都&startDate=2026-03-16&endDate=2026-04-16&flightType=全部
```

**响应**:
```json
{
  "destinations": [
    {
      "destination": "广州",
      "flightCount": 7,
      "availableDates": ["2026-03-17", "2026-03-18", ...],
      "cardTypes": ["666权益卡航班", "2666权益卡航班"],

    "end": "2026-04-16"
  }
}
```

## 💡 使用场景

### 场景 1：周末短途游
用户想在周末去广州玩，需要周五晚上出发，周日晚上返回：
- 查询北京到广州的目的地
- 看到"可往返"标签，确认有返程航班
- 点击查看详情，选择合适的往返航班组合

### 场景 2：长假旅行规划
用户计划国庆假期去多个城市，需要确认每个城市是否支持往返：
- 快速浏览目的地列表的"可往返"标签
- 筛选出支持往返的城市
- 规划合理的多城市行程

### 场景 3：灵活行程安排
用户不确定返程日期，想看看返程航班的选择：
- 查看返程航班的可用日期
- 对比不同日期的航班时间
- 选择最方便的返程日期

## ✨ 总结

往返航班查询功能大大提升了用户体验，使得行程规划更加便捷和直观。通过在目的地列表中直接展示往返信息，用户可以快速做出决策，节省了大量的查询时间。

**核心价值**：
- 🎯 **提高效率**：一次查询获取完整往返信息
- 👀 **直观展示**：清晰的视觉标识和分段展示
- 💡 **智能推荐**：自动识别支持往返的目的地
