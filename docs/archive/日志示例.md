## 🎯 日志内容说明

### 基本字段

| 字段 | 说明 | 示例值 |
|------|------|--------|
| `id` | 主键 | 1 |
| `taskType` | 任务类型 | discover_airports / refresh_flights / full_initialize |
| `status` | 执行状态 | running / success / failed |
| `days` | 爬取天数 | 1, 7, 30 |
| `airportCount` | 机场数量 | 15 |
| `flightCount` | 航班数量 | 3850 |
| `startTime` | 开始时间 | 2026-03-16 10:00:00 |
| `endTime` | 结束时间 | 2026-03-16 10:02:15 |
| `duration` | 执行时长（秒） | 135 |
| `createdAt` | 创建时间 | 2026-03-16 10:00:00 |

### 详细信息字段（details）

#### 发现机场
- `seedAirports`: 种子机场列表
- `seedAirportResults`: 每个种子机场发现的航班数
- `discoveredAirports`: 发现的所有机场
- `dateRange`: 爬取的日期范围

#### 发现航班
- `airports`: 爬取的机场列表
- `airportResults`: 每个机场爬取的航班数
- `dateRange`: 爬取的日期范围
- `deletedCount`: 删除的旧航班数量

### 错误信息字段（errorMessage）

失败时记录的错误信息，例如：
- `"Timeout: Navigation timeout of 30000 ms exceeded"` - 页面加载超时
- `"Network error: Failed to fetch"` - 网络错误
- `"未知错误"` - 其他未捕获的错误

1. ✅ **完整的执行信息**：任务类型、状态、天数、机场数、航班数
2. ✅ **时间追踪**：开始时间、结束时间、执行时长（自动计算）

   - 发现机场：种子机场、每个机场的结果、发现的机场列表、日期范围

4. ✅ **错误追踪**：失败时记录详细的错误信息
5. ✅ **可查询性**：可按任务类型、状态筛选，按时间排序
6. ✅ **可追溯性**：每次执行都有唯一 ID，可查看完整历史
