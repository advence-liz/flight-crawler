## 根本原因分析

### 为什么会出现这个问题？

1. **需求理解不完整**
   - 假设：只有海南航空运营特价航班
   - 实际：多家航空公司都运营特价航班

2. **代码审查不足**
   - 正则表达式硬编码了特定航空公司
   - 没有考虑可扩展性

3. **测试覆盖不足**
   - 没有为不同航空公司的航班号编写测试

### 为什么没有立即发现？

```
问题隐蔽性：
├─ 爬虫表面上"成功"运行（没有报错）
├─ 数据库里有数据（只是不完整）
├─ 只有特定机场受影响
└─ 需要手动对比网站才能发现

影响：
└─ 用户看到北京大兴有 0 条航班
   但网站上明确显示有航班
```

## 修复策略

### 选择的方案：通用正则表达式

```
方案对比：

方案 A: 硬编码所有航空公司代码 ❌
├─ 代码：/HU|JD|PN|CA|MU|CZ\d{4}/
├─ 优点：精确
└─ 缺点：需要维护列表，不可扩展

方案 B: 通用正则表达式 ✅ (选中)
├─ 代码：/[A-Z]{2}\d{4}/
├─ 优点：简洁、可扩展、符合 IATA 标准
└─ 缺点：可能误匹配其他数据（但概率很低）

方案 C: 从网站提取航空公司代码 ❌
├─ 代码：需要额外的 DOM 解析
├─ 优点：最准确
└─ 缺点：复杂度高，维护成本大
```
