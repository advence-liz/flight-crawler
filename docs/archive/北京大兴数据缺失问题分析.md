### 2. 问题定位

**问题不在于操作流程，而在于数据提取阶段！**

所有操作步骤都成功了：
- ✅ 权益卡选择成功
- ✅ 出发地输入成功
- ✅ 城市选择成功
- ✅ 查询按钮点击成功

但最终提取到的航班数据为 0 条。

### 3. 可能的原因

#### 原因 1：页面加载时间不足 ⭐⭐⭐⭐⭐

**最可能的原因**：点击查询按钮后，等待时间不够，页面数据还没有完全加载就开始提取数据。

**证据**：
- 深圳爬取成功（28 条航班）
- 其他种子机场（北京首都、北京大兴、上海浦东、上海虹桥）都是 0 条

这说明可能是并发爬取导致的资源竞争，或者页面加载速度不一致。

#### 原因 2：城市名称匹配问题 ⭐⭐⭐

**可能性**：在下拉列表中选择城市时，可能选择了错误的选项。

**分析**：
- 日志显示"✅ 选择城市成功: 北京大兴"
- 但这只是说找到了包含"北京大兴"的选项并点击了
- 实际可能选择了"北京大兴国际机场"而不是"北京大兴"

#### 原因 3：数据提取选择器不匹配 ⭐⭐⭐⭐

**可能性**：页面结构发生了变化，或者北京大兴的页面结构与其他城市不同。

**分析**：
- 深圳能提取到数据，说明选择器对某些城市是有效的
- 但北京大兴、北京首都等都提取不到，可能是页面结构不同

#### 原因 4：网站反爬机制 ⭐⭐

**可能性**：并发爬取触发了网站的反爬机制。

**证据**：
- 5 个种子机场并发爬取
- 只有深圳成功，其他都失败
- 可能深圳是第一个完成的，后续请求被限制

## 🎯 推荐解决方案

### 方案 1：增加等待时间（优先级：高）

**修改位置**：`crawler.service.ts` 的 `crawlFlightsByDate` 方法

**修改内容**：
```typescript
// 点击查询按钮后
await this.clickSearchButton(page);

// 增加等待时间
await page.waitForTimeout(5000); // 从 3000 增加到 5000

// 或者使用智能等待
try {
  await page.waitForSelector('.flight-list, .no-data, .error-message', {
    timeout: 10000
  });
} catch (error) {
  this.logger.warn('等待航班列表超时，继续尝试提取数据');
}
```

### 方案 2：禁用并发爬取（优先级：中）

**修改位置**：`crawler.service.ts` 的 `initializeDiscoverAirports` 方法

**修改内容**：
```typescript
// 当前：并行爬取
const crawlPromises = seedAirports.map(airport =>
  this.crawlFlights(airport, dates, true, false)
);
const flightsArrays = await Promise.all(crawlPromises);

// 修改为：顺序爬取
const flightsArrays = [];
for (const airport of seedAirports) {
  try {
    const { flights } = await this.crawlFlights(airport, dates, true, false);
    this.logger.log(`✅ ${airport} 发现 ${flights.length} 条航班`);
    seedAirportResults[airport] = flights.length;
    flightsArrays.push(flights);
  } catch (error) {
    this.logger.error(`❌ ${airport} 爬取失败`, error);
    seedAirportResults[airport] = 0;
    flightsArrays.push([]);
  }

  // 每个城市之间添加延迟
  if (seedAirports.indexOf(airport) < seedAirports.length - 1) {
    await new Promise(resolve => setTimeout(resolve, 2000));
  }
}
```

### 方案 3：改进数据提取逻辑（优先级：高）

**修改位置**：`crawler.service.ts` 的数据提取部分

**修改内容**：
```typescript

  retryCount++;
}
```
