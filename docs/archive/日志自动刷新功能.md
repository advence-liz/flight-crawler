# 日志自动刷新功能说明

## 🎯 功能概述

为执行日志页面添加了可手动开启的自动刷新功能，方便实时监控爬虫任务的执行状态。

---

## ✨ 核心功能

### 1. 自动刷新开关 ✅

**位置**：执行日志卡片右上角（统计信息旁边）

**功能**：
- ✅ 一键开启/关闭自动刷新
- ✅ 开启时按钮变为主色调（蓝色）
- ✅ 图标旋转动画（开启时）
- ✅ 显示当前刷新间隔

**按钮状态**：
- 关闭：`[自动刷新]` - 灰色按钮，静态图标
- 开启：`[自动刷新中 (5s)]` - 蓝色按钮，旋转图标

### 2. 刷新间隔设置 ✅

**位置**：日志筛选区域（任务类型和状态筛选器旁边）

**功能**：
- ✅ 仅在开启自动刷新时显示
- ✅ 支持 5 种刷新间隔：3秒、5秒、10秒、30秒、60秒
- ✅ 默认 5 秒
- ✅ 切换间隔时立即生效

**可选间隔**：
- 3 秒 - 快速监控（执行中的任务）
- 5 秒 - 默认（推荐）
- 10 秒 - 常规监控
- 30 秒 - 低频监控
- 60 秒 - 定期检查

### 3. 自动刷新内容 ✅

**刷新内容**：
- ✅ 日志列表（当前页）
- ✅ 统计信息（总执行次数、成功次数、失败次数、今日执行次数）

**不刷新内容**：
- 筛选条件（保持用户选择）
- 分页状态（保持当前页）
- 日志详情 Modal（如果打开）

### 4. 消息提示 ✅

**开启自动刷新**：
```
✅ 已开启自动刷新（每 5 秒）
```

**关闭自动刷新**：
```
ℹ️ 已关闭自动刷新
```

**切换刷新间隔**：
```
ℹ️ 刷新间隔已设置为 10 秒
```

## 📦 技术实现

### 前端实现

**文件**: `frontend/src/pages/DataManagement.tsx`

#### 1. 状态管理

```typescript
const [autoRefresh, setAutoRefresh] = useState(false); // 是否开启自动刷新
const [refreshInterval, setRefreshInterval] = useState(5); // 刷新间隔（秒）
const autoRefreshTimerRef = useRef<NodeJS.Timeout | null>(null); // 定时器引用
```

#### 2. 自动刷新逻辑

```typescript
useEffect(() => {
  // 清除旧的定时器
  if (autoRefreshTimerRef.current) {
    clearInterval(autoRefreshTimerRef.current);
    autoRefreshTimerRef.current = null;
  }

  // 如果开启自动刷新，设置新的定时器
  if (autoRefresh) {
    autoRefreshTimerRef.current = setInterval(() => {
      loadLogs(logsPagination.current, logsPagination.pageSize);
      loadLogStats();
    }, refreshInterval * 1000);
  }

  // 清理函数（组件卸载时清除定时器）
  return () => {
    if (autoRefreshTimerRef.current) {
      clearInterval(autoRefreshTimerRef.current);
      autoRefreshTimerRef.current = null;
    }
  };
}, [autoRefresh, refreshInterval, logsPagination.current, logsPagination.pageSize]);
```

#### 3. 切换自动刷新

```typescript
const toggleAutoRefresh = () => {
  setAutoRefresh(!autoRefresh);
  if (!autoRefresh) {
    message.success(`已开启自动刷新（每 ${refreshInterval} 秒）`);
  } else {
    message.info('已关闭自动刷新');
  }
};
```

#### 4. 自动刷新按钮

```tsx
<Button
  type={autoRefresh ? 'primary' : 'default'}
  icon={<SyncOutlined spin={autoRefresh} />}
  onClick={toggleAutoRefresh}
>
  {autoRefresh ? `自动刷新中 (${refreshInterval}s)` : '自动刷新'}
</Button>
```

#### 5. 刷新间隔选择器

```tsx
{autoRefresh && (
  <Select
    value={refreshInterval}
    style={{ width: 120 }}
    onChange={(value) => {
      setRefreshInterval(value);
      message.info(`刷新间隔已设置为 ${value} 秒`);
    }}
  >
    <Select.Option value={3}>3 秒</Select.Option>
    <Select.Option value={5}>5 秒</Select.Option>
    <Select.Option value={10}>10 秒</Select.Option>
    <Select.Option value={30}>30 秒</Select.Option>
    <Select.Option value={60}>60 秒</Select.Option>
  </Select>
)}
```

## 💡 使用场景

### 场景 1：监控正在执行的任务

**需求**：执行"发现航班"（30 天），需要实时监控进度

**操作**：
1. 执行"发现航班"任务
2. 切换到"执行日志"卡片
3. 点击"自动刷新"按钮
4. 选择刷新间隔为"3 秒"（快速监控）
5. 观察日志状态变化（执行中 → 成功/失败）

**效果**：
- 每 3 秒自动刷新日志列表
- 实时看到任务状态变化
- 无需手动刷新页面

### 场景 2：定期检查执行历史

**需求**：查看今日的执行情况，定期检查是否有新的执行记录

**操作**：
1. 访问数据管理页面
2. 点击"自动刷新"按钮
3. 使用默认间隔（5 秒）或选择"10 秒"
4. 继续其他工作

**效果**：
- 自动刷新日志列表
- 发现新的执行记录
- 统计信息实时更新

### 场景 3：长时间监控

**需求**：需要长时间监控系统运行情况，但不希望频繁刷新

**操作**：
1. 开启自动刷新
2. 选择刷新间隔为"30 秒"或"60 秒"
3. 最小化窗口或切换到其他标签页

**效果**：
- 低频率刷新，减少服务器压力
- 长时间监控不会过度消耗资源
- 回到页面时看到最新数据

### 场景 4：调试爬虫问题

**需求**：爬虫频繁失败，需要快速查看错误信息

**操作**：
1. 开启自动刷新
2. 选择刷新间隔为"3 秒"
3. 筛选状态为"失败"
4. 观察失败日志

**效果**：
- 快速发现新的失败记录
- 及时查看错误信息
- 快速定位问题

## ⚠️ 注意事项

### 1. 性能考虑

- ⚠️ 刷新间隔过短（如 3 秒）会增加服务器压力
- 建议使用默认的 5 秒间隔
- 长时间监控建议使用 30 秒或 60 秒

### 2. 自动清理

- 组件卸载时会自动清除定时器
- 关闭自动刷新时会立即清除定时器
- 切换刷新间隔时会重新创建定时器

### 3. 用户体验

- 开启自动刷新时，按钮变为蓝色，图标旋转
- 刷新间隔选择器仅在开启时显示
- 提供明确的消息提示

### 4. 数据一致性

- 自动刷新不会改变用户的筛选条件
- 保持当前页码和每页数量
- 不会打断用户正在查看的日志详情
