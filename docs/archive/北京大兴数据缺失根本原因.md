# 北京大兴数据缺失根本原因

## 🔴 重大发现：日期选择逻辑缺失

### 问题描述

在分析"发现机场"任务时，发现**所有机场都返回 0 条航班**的根本原因：

**爬虫代码中完全没有实现日期选择逻辑！**

### 代码分析

#### 当前代码流程（有问题）

```typescript
// crawler.service.ts - crawlFlightsByDate 方法

async crawlFlightsByDate(origin: string, date: string, cardTypes: string | string[]) {
  // 1. ✅ 访问页面
  await page.goto('https://m.hnair.com/hnams/plusMember/ableAirlineQuery');

  // 2. ✅ 选择权益卡类型
  await this.selectCardType(page, cardType);

  // 3. ✅ 选择出发地
  await this.selectOrigin(page, origin);

  // 4. ❌ 缺失：选择日期
  // 这里应该有：await this.selectDate(page, date);

  // 5. ✅ 点击查询按钮
  await this.clickSearchButton(page);

  // 6. ✅ 提取数据
  const flights = await this.extractFlightData(page);
}
```

**关键问题**：虽然方法接收了 `date` 参数，但从未在页面上选择这个日期！

#### 实际执行情况

### 用户反馈验证

用户说："**切换日期后就有数据了**"

**爬虫代码从未切换日期，所以一直查询的是没有数据的明天！**

## ✅ 解决方案

### 1. 新增 `selectDate` 方法

```typescript
/**
 * 选择出发日期
 */
private async selectDate(page: puppeteer.Page, targetDate: string): Promise<boolean> {
  this.logger.log(`📅 尝试选择日期: ${targetDate}`);

  try {
    // 策略 1: 点击日期选择器
    const dateBoxClicked = await page.evaluate(() => {
      const selectors = [
        '.date-box',
        '.dep-date',
        '[class*="date"]',
        'input[type="date"]',
        '[placeholder*="日期"]',
        '[placeholder*="出发"]',
      ];

      for (const selector of selectors) {
        const elements = document.querySelectorAll(selector);
        for (const el of elements) {
          const text = (el.textContent || '').trim();
          if (text.includes('日期') || text.includes('出发') || text.match(/\d{4}-\d{2}-\d{2}/)) {
            (el as HTMLElement).click();
            return true;
          }
        }
      }
      return false;
    });

    if (!dateBoxClicked) {
      this.logger.warn('⚠️ 未找到日期选择器，可能页面默认日期已满足需求');
      return true;
    }

    await page.waitForTimeout(1000);

    // 策略 2: 从日历中选择日期
    const targetDateObj = new Date(targetDate);
    const targetDay = targetDateObj.getDate();

    const dateSelected = await page.evaluate((day) => {
      const selectors = [
        '.calendar-day',
        '.date-item',
        '[class*="day"]',
        '[class*="date-cell"]',
      ];

      for (const selector of selectors) {
        const elements = document.querySelectorAll(selector);
        for (const el of elements) {
          const text = (el.textContent || '').trim();
          const dayNumber = parseInt(text, 10);

          if (dayNumber === day) {
            const isDisabled = el.classList.contains('disabled') ||
                               el.classList.contains('past');

            if (!isDisabled) {
              (el as HTMLElement).click();
              return true;
            }
          }
        }
      }
      return false;
    }, targetDay);

    if (dateSelected) {
      this.logger.log(`✅ 日期选择成功: ${targetDate}`);
      await page.waitForTimeout(1000);
      return true;
    }

    // 策略 3: 尝试直接输入日期
    const dateInput = await page.$('input[type="date"], input[placeholder*="日期"]');
    if (dateInput) {
      await dateInput.click({ clickCount: 3 });
      await page.keyboard.type(targetDate);
      await page.keyboard.press('Enter');
      this.logger.log(`✅ 日期输入成功: ${targetDate}`);
      await page.waitForTimeout(1000);
      return true;
    }

    this.logger.warn('⚠️ 所有日期选择策略都失败，使用页面默认日期');
    return true;

  } catch (error) {
    this.logger.error(`❌ 选择日期异常: ${error.message}`);
    return false;
  }
}
```

## 📝 总结

### 根本原因

**爬虫代码中缺失日期选择逻辑**，导致：
1. 虽然循环遍历了7天的日期
2. 但每次查询都使用页面默认日期（明天）
3. 如果明天没有数据，就会返回 0 条航班
4. 完全没有查询后续6天的数据

### 用户反馈验证

用户说"切换日期后就有数据了"，正好印证了：
- 明天（默认日期）：没有数据
- 其他日期：有数据

### 解决方案

1. ✅ 新增 `selectDate` 方法（多种策略）
2. ✅ 在 `crawlFlightsByDate` 中调用日期选择
3. ✅ 添加日期选择失败的截图和日志
