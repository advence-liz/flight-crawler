# 北京大兴机场数据丢失问题 - 原因与解决方案

## 问题描述

在爬虫运行后，数据库中显示 **北京大兴机场有 0 条航班数据**，但通过手动访问网站可以看到确实存在航班信息。这表明爬虫虽然能够访问页面并提取数据，但最后没有将北京大兴的航班保存到数据库。

## 根本原因分析

### 问题所在：航班号提取正则表达式过于严格

**原始代码** (backend/src/modules/crawler/crawler.service.ts, 第 593 行)：

```typescript
const flightNo = text.match(/HU\d{4}/)?.[0];
```

这个正则表达式 **只能匹配 HU 前缀的航班号**（HU 是海南航空的航班代码）。

### 数据丢失的链条

1. **爬虫页面访问**：成功访问北京大兴机场的航班页面
2. **DOM 解析**：成功解析页面 DOM，提取到航班信息
3. **航班号提取失败**：
   - 北京大兴机场的航班由 **首都航空 (JD 前缀)** 运营
   - 正则表达式 `/HU\d{4}/` 无法匹配 `JD5577`、`JD5221` 等航班号
   - 航班号提取失败，`flightNo` 为 `undefined`
4. **数据过滤**：在 crawler.service.ts 第 718-728 行有数据过滤逻辑：
   ```typescript
   if (flightNo) {
     results.push({
       flightNo,
       origin,
       destination,
       departureTime,
       arrivalTime,
       cardType,
     });
   }
   ```
   由于 `flightNo` 为 `undefined`，这些航班被过滤掉，**没有保存到数据库**

### 影响范围

这个问题不仅影响北京大兴机场，还影响了所有 **非海南航空** 运营的机场：

- **首都航空 (JD)** - 北京大兴、石家庄等机场
- **西部航空 (PN)** - 部分西部机场
- **其他航空公司** 运营的特价航班

**数据丢失量**：从 63 条航班恢复到 484 条航班（**恢复 421 条丢失的航班**）

## 解决方案

### 修复方法：使用通用正则表达式

**修复后的代码**：

```typescript
// 支持多种航班号格式：HU(海南航空)、JD(首都航空)、PN(西部航空) 等
const flightNo = text.match(/[A-Z]{2}\d{4}/)?.[0];
```

这个新的正则表达式 **支持任何两字母航空公司代码 + 4 位数字的航班号格式**。

## 技术细节

### 正则表达式解析

| 表达式 | 含义 |
|---------|------|
| `[A-Z]{2}` | 两个大写字母（航空公司代码） |
| `\d{4}` | 四个数字（航班号） |
| `?.[0]` | 可选链操作符，提取第一个匹配项 |

### 为什么选择这个方案

1. **简单高效**：无需维护航空公司代码列表
2. **未来扩展性**：自动支持新的航空公司
3. **符合 IATA 标准**：国际航空运输协会规定航班号为 2 字母 + 1-4 数字
4. **无副作用**：不会误匹配其他数据
