## 🎯 核心改进

### 1. 独立管理界面 ✅

**改动前**：
- 数据管理按钮混杂在目的地查询页面的 Header 中
- 功能不够突出和专业

**改动后**：
- 创建独立的"数据管理"页面（`/data-management`）
- 新增导航菜单项，与其他功能平级
- 专业的管理界面，清晰的功能分区

### 2. 自定义时间范围 ✅

**改动前**：
- 时间范围硬编码在后端
- 发现机场：固定 1 天
- 发现航班：固定 7 天

**改动后**：
- 支持前端传递参数
- 发现机场：可选 1-7 天（默认 1 天）
- 发现航班：可选 1-30 天（默认 7 天）
- 参数可选，不传则使用默认值（向后兼容）

### 3. 可视化流程 ✅

- 使用 Ant Design Steps 组件展示执行流程
- 清晰的三步流程：发现机场 → 发现航班 → 完成
- 图标和描述说明每个步骤的作用

### 4. 实时结果反馈 ✅

- 发现机场：显示机场数量和临时航班数
- 发现航班：显示爬取的航班总数
- 一键初始化：显示总计航班数
- 使用 Statistic 组件展示统计数据

### 5. 使用指导 ✅

- 顶部使用说明 Alert
- 底部注意事项 Alert
- 每个功能卡片都有详细说明
- 表单字段有 extra 提示文字

## 📦 技术实现

### 后端改动（3 个文件）

#### 1. 新增 DTO 验证
**文件**: `backend/src/modules/crawler/dto/initialize.dto.ts`

```typescript
export class InitializeDiscoverDto {
  @IsOptional()
  @IsInt()
  @Min(1)
  @Max(7)
  days?: number = 1;
}

export class InitializeRefreshDto {
  @IsOptional()
  @IsInt()
  @Min(1)
  @Max(30)
  days?: number = 7;
}
```

#### 2. 更新 Controller
**文件**: `backend/src/modules/crawler/crawler.controller.ts`

- 导入 `@Body()` 装饰器和 DTO
- 接收并传递参数到 Service 层

#### 3. 更新 Service
**文件**: `backend/src/modules/crawler/crawler.service.ts`

- `initializeDiscoverAirports(days: number = 1)`
- `initializeRefreshFlights(days: number = 7)`
- 根据参数动态生成日期范围

## 🔍 API 示例

### 1. 使用默认值

```bash
# 发现机场（默认 1 天）
curl -X POST http://localhost:3000/api/crawler/initialize/discover \
  -H "Content-Type: application/json" \
  -d '{}'

# 发现航班（默认 7 天）
curl -X POST http://localhost:3000/api/crawler/initialize/refresh \
  -H "Content-Type: application/json" \
  -d '{}'
```

### 2. 自定义天数

```bash
# 发现机场（3 天）
curl -X POST http://localhost:3000/api/crawler/initialize/discover \
  -H "Content-Type: application/json" \
  -d '{"days": 3}'

# 发现航班（30 天）
curl -X POST http://localhost:3000/api/crawler/initialize/refresh \
  -H "Content-Type: application/json" \
  -d '{"days": 30}'
```
