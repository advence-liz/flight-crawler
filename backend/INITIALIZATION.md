# 系统初始化说明

## 设计理念

系统采用**两阶段初始化**策略，自动发现和管理机场数据，无需手动配置机场列表。

### 为什么需要两阶段初始化？

**问题**：同一城市可能有多个机场（如北京首都、北京大兴），如果只用城市名爬取会遗漏数据。

**解决方案**：
1. **阶段1（发现）**：从种子城市出发，自动发现所有可达的机场
2. **阶段2（刷新）**：基于发现的机场，全量爬取航班数据

## 初始化流程

### 阶段1：发现机场 🔍

**接口**: `POST /api/crawler/initialize/discover`

**功能**：
- 从种子城市（北京、上海、深圳）出发
- 爬取明天的航班数据（快速发现）
- 自动提取所有出发地和目的地机场
- 保存到 `airports` 表

**示例**：
```bash
curl -X POST http://localhost:3000/api/crawler/initialize/discover
```

**返回**：
```json
{
  "success": true,
  "airportCount": 15,
  "flightCount": 42,
  "message": "机场发现完成！发现 15 个机场，42 条航班"
}
```

**发现的机场示例**：
- 北京首都、北京大兴（如果有航班）
- 上海浦东、上海虹桥
- 深圳
- 以及所有从种子机场可达的其他机场（如长沙、广州、成都天府等）

**重要概念**：
- 所有机场都可以作为出发地或目的地，不做区分
- 种子机场只是为了快速发现网络中的所有机场
- 发现后，所有机场都可以用于后续的航班查询和路径规划

### 阶段2：刷新航班数据 🔄

**接口**: `POST /api/crawler/initialize/refresh`

**功能**：
- 读取 `airports` 表中所有启用爬虫的出发地机场
- 为每个机场爬取未来 7 天的航班数据
- 保存到 `flights` 表

**示例**：
```bash
curl -X POST http://localhost:3000/api/crawler/initialize/refresh
```

**返回**：
```json
{
  "success": true,
  "count": 350,
  "message": "数据刷新完成！共爬取 350 条航班"
}
```

### 一键初始化 ⚡

**接口**: `POST /api/crawler/trigger`

**功能**：
- 自动判断是否首次运行
- 如果是首次：执行阶段1 → 阶段2
- 如果已有数据：直接执行阶段2

**示例**：
```bash
curl -X POST http://localhost:3000/api/crawler/trigger
```

## 前端使用

在目的地查询页面，提供三个按钮：

1. **发现机场**：执行阶段1，发现所有机场（首次使用时点击）
2. **刷新数据**：执行阶段2，更新航班数据（日常更新时点击）
3. **一键初始化**：智能判断，自动执行完整流程（推荐）

## 定时任务

系统会在每天凌晨 2 点自动执行：
1. 调用 `triggerCrawl()` 更新数据
2. 清理过期航班数据

## 机场管理

### 查看已发现的机场

```bash
# 查看所有出发地机场
curl http://localhost:3000/api/flights/cities | jq '.origins'

# 查看所有目的地机场
curl http://localhost:3000/api/flights/cities | jq '.destinations'
```

### 机场数据结构

```typescript
{
  name: '北京首都',      // 完整机场名称
  city: '北京',          // 城市名称
  enableCrawl: true,     // 是否启用爬虫（所有机场都可作为出发地或目的地）
  discoveredAt: Date,    // 发现时间
}
```

## 优势

1. **自动发现**：无需手动维护机场列表配置
2. **不遗漏数据**：确保同城多机场都能被爬取（如北京首都、北京大兴）
3. **灵活扩展**：新增航线会自动发现新机场
4. **清晰展示**：前端明确显示完整机场名称，无歧义

## 注意事项

1. **首次使用**：必须先执行"发现机场"或"一键初始化"
2. **数据更新**：建议每天执行"刷新数据"或依赖定时任务
3. **机场名称**：系统会保留完整机场名称（如"北京首都"，不会简化为"北京"）
4. **种子机场**：使用北京首都、北京大兴、上海浦东、上海虹桥、深圳作为种子，确保同城多机场都能被覆盖
5. **机场平等性**：所有发现的机场都可以作为出发地或目的地，不做区分
